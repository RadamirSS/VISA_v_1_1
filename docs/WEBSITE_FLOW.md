# Website Flow

Сайт используется только как lead-generation слой.

## Пользовательский путь

1. Пользователь читает информацию о сервисе и стране подачи.
2. Пользователь оставляет короткую заявку.
3. Сайт валидирует обязательные поля и хотя бы один контакт.
4. API `POST /api/leads` принимает данные.
5. Если заданы `LEADS_TELEGRAM_BOT_TOKEN` и `LEADS_TELEGRAM_CHAT_ID`, менеджеру уходит Telegram-уведомление.
6. Если env-переменные не заданы, сайт не падает и все равно показывает успешный ответ.

## Что сайт не делает

- не собирает паспортные данные
- не собирает файлы
- не выдает себя за официальный визовый портал
- не обещает запись или одобрение визы
