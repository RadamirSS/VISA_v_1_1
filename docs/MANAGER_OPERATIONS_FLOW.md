# Manager Operations Flow

Операционная консоль менеджера в Telegram-боте для Mini App visa cases. Это не web dashboard — менеджер работает вручную через бот.

## Быстрый маршрут

```
📥 Новые заявки
  → выбрать кейс [Открыть]
  → сводка кейса
  → действие:
      📋 Анкеты
      📎 Документы
      📅 Отправить даты
      ✅ Подтвердить запись
      🔄 Статус
      ✉️ Сообщение клиенту
      ↩️ Назад в очередь
```

Поиск: **🔎 Найти заявку** — принимает `VISA-2026-*` (старые заявки), `VISA-CASE-*`, UUID, `telegram_id`, `@username`.

## Очередь менеджера

`📥 Новые заявки` группирует активные Mini App кейсы по операционным этапам:

| Группа | Когда попадает |
|--------|----------------|
| Новые на проверку | `submitted_for_manager_review`, `needs_manager_consultation`, `waiting_manager_review` |
| Нужны документы | `manager_reviewing`, `needs_clarification` |
| Ожидаем документы клиента | есть client-required документы в статусе `requested` |
| Документы на проверке | есть документы в статусе `uploaded_by_client` |
| Подбираем даты | `ready_for_slot_search` |
| Даты отправлены | `slot_options_sent` |
| Клиент выбрал дату | `slot_selected_by_client`, `appointment_confirmation_pending` |
| Запись подтверждена | `appointment_confirmed` |

Каждый элемент компактный: public number, страна/город, число заявителей, человекочитаемый статус, кнопка **Открыть**.

Сводка очереди показывает все активные кейсы по группам с человекочитаемыми статусами.

Отдельные карточки с кнопкой **Открыть** отправляются только для первых 10 уникальных кейсов. Если кейсов больше — в сводке появляется сообщение:

`Показаны первые 10 кейсов. Для поиска используйте 🔎 Найти заявку.`

То же поведение у **📥 Новые заявки** и **↩️ Назад в очередь** (`mgrcase:back`).

Старые order-based заявки (`VISA-2026-*`) отображаются отдельным блоком внизу — их flow не изменён.

## Case workspace (сводка кейса)

При открытии кейса менеджер видит безопасную сводку:

- public number (`VISA-CASE-YYYY-XXXXXX`)
- internal case id
- клиент: `@username` / `telegram_id`
- страна, город подачи, визовый центр, цель поездки
- число заявителей и прогресс анкет (`X/Y заполнено`)
- документы: `N ожидают / M на проверке` (клиент) и `N в работе / M готов` (агентство)
- запись: даты отправлены / дата выбрана / подтверждена
- статус на русском языке

**Не показывается:** номер паспорта, место рождения, адрес, банковские данные, пути к файлам, ссылки на скачивание, полные анкеты.

## Действия из workspace

### 📋 Анкеты

Список заявителей: имя (или роль), процент заполнения, статус (`заполнено`, `черновик` и т.д.). Без паспортных и адресных полей.

### 📎 Документы

Открывает существующий document flow **без повторного ввода case ID**. Доступны все прежние действия: показать, запросить у клиента, добавить агентство, загрузить файл, изменить статус, комментарий.

Подробнее: [DOCUMENT_REQUEST_CHECKLIST.md](DOCUMENT_REQUEST_CHECKLIST.md)

### 📅 Отправить даты

Запускает slot offer flow **без повторного ввода case ID**. Формат ввода:

```
2026-07-15 10:30
2026-07-16 14:00 | Москва | VMS Italy | Комментарий
```

Подробнее: [MANAGER_SLOT_OFFERS_FLOW.md](MANAGER_SLOT_OFFERS_FLOW.md)

### ✅ Подтвердить запись

Подтверждает запись для текущего кейса (требуется выбранный клиентом слот).

### 🔄 Статус

Менеджер может установить:

- Менеджер проверяет данные
- Требуются уточнения
- Менеджер подбирает даты
- Менеджер отправил варианты дат
- Дата выбрана клиентом
- Запись подтверждается
- Запись подтверждена
- Заявка отменена / закрыта

**Guards:**

- нельзя подтвердить запись без выбранного слота
- нельзя поставить «даты отправлены» без созданных slot options
- отменённые/закрытые кейсы не показывают активные действия

### ✉️ Сообщение клиенту

Безопасные шаблоны: проверяем заявку, нужны уточнения, запросили документы, ожидаем документы, документы получены, подбираем даты, даты отправлены, дата выбрана, запись подтверждена, свяжитесь с менеджером.

Текст не содержит чувствительных данных.

## Callback namespace

```
mgrcase:open:{case_id}
mgrcase:applicants:{case_id}
mgrcase:documents:{case_id}
mgrcase:slots:{case_id}
mgrcase:confirm:{case_id}
mgrcase:status:{case_id}
mgrcase:setstatus:{case_id}:{status}
mgrcase:message:{case_id}
mgrcase:template:{case_id}:{code}
mgrcase:back
```

## Что намеренно не включено

- полноценный web admin dashboard
- реальная booking API integration
- онлайн-платежи
- публичные ссылки на файлы
- PDF generation
- новые поля клиентской анкеты
