# Telegram Bot Flow

## Приоритетный маршрут

После активации ключа доступа бот ведет клиента в Mini App:

1. клиент активирует ключ
2. открывает `📋 Открыть личный кабинет`
3. заполняет анкеты заявителей
4. создает визовый кейс
5. выбирает страну, город подачи и визовый центр
6. отправляет заявку менеджеру
7. получает варианты дат записи
8. выбирает дату
9. получает подтверждение записи
10. получает запрос документов и открывает Mini App → **Документы**

## Manager document actions

Меню: **📎 Документы по заявке**

- запросить документ у клиента (шаблоны: паспорт, фото, выписка, страховка, справка с работы, другое)
- добавить документ, который готовит агентство
- изменить статус документа
- оставить комментарий клиенту

Telegram-уведомления по документам содержат только metadata (название документа, case id, статус). Без файлов, ссылок и чувствительных данных.

Подробнее: [DOCUMENT_REQUEST_CHECKLIST.md](DOCUMENT_REQUEST_CHECKLIST.md)

## Что остается в боте

- `/start`, `/menu`, `/help`, `/status`, `/cancel`, `/admin`
- consent и регистрация
- ввод ключа доступа
- контакт с менеджером
- существующий appointment FSM как legacy fallback

## Что нельзя отправлять через чат

- номер паспорта
- срок действия паспорта
- место рождения
- адрес проживания
- содержимое полной анкеты
- банковские данные

## Manager notifications

Бот отправляет только safe summary:

- completion notification по анкетам
- submitted case notification по визовой заявке
- slot offer notification с вариантами дат
- client selection notification для менеджеров
- final appointment confirmation notification для клиента

Оба сообщения не должны содержать чувствительные поля.
